Hands-on Activity

Enough talk, let's start doing stuff. This tutorial will be using Git solely from the command line. If you have any questions, don't hesitate to ask me. I repeat, if you have any questions, please ask me!

In this tutorial, you'll be adding your name to this notebook!

  1. Create a GitHub account for this, if not already. Once you do, let me know so I can add you as a collaborator to this repository.
  2. Install Git. You can either install it from their website, or (if you're on Windows) you can download Ubuntu from the Windows App Store and use WSL. If you're on Mac, Git is usually preinstalled in the terminal. Obviously there are many other ways to do this.
  3. Verify Git is installed in your terminal by typing$ git --versionYour terminal should show what version of Git you have installed.
  4. Now navigate to the directory (aka folder) you want to store the sample repository in. From your current directory (which usually is your home directory), you can type$ lsto show all the files and subdirectories, and$ cd <subdirectory name>to enter that subdirectory. To go back a directory, type "cd ..". The two periods means to go back.
  5. Now clone this repository using$ git clone https://github.com/samuel-ping/intro-to-git.git
  6. If you type the "ls" command again, you'll see this intro-to-git subdirectory! Enter this subdirectory using the "cd" command.
  7. Using your favorite code editor (if you have VSCode installed, you should be able to type "code ." to open the directory in VSCode), go into the examples/ directory, and edit the message_book.html file to include your message!
  8. You're not done yet! Now that you added your message, if you type$ git statusyou'll see that Git tells you that you modified the message book file.
  9. Now add your changed file to the staging area using$ git add examples/message_book.htmlIf you're already in the examples/ directory, you can omit that part from your command.
  10. If you type "git status" again, you'll see that the file is now staged for commit! Now commit the file using$ git commit -m "<message stating reason for commit>"
  11. You can now push your changes back to the repository on GitHub. To do this, type$ git push origin master
  12. If you go to the repository, you should see your changes pop up!

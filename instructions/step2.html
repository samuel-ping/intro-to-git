Git Commands

Now that you know how Git flows, let's compare each step to its respective command.

Git Commands

  1. $ git clone
  2. $ git add <file name>
  3. $ git commit -m "<message stating reason for commit>"
  4. $ git push

Steps in English

  1. You create a repo.
  2. When you make changes to your repository, you then add it to the staging area. You can do this as many times as you want.
  3. Once you've added all of your modified files to the staging area, you commit those changes with a message stating what the changes are. You can only commit when there are files in the staging area that you added in the previous step. After committing, your files are removed from the staging area.
  4. Once you've made your commit(s), you then push your changes to your repository. This is the part that actually updates the repository, whether it be hosted on GitHub or somewhere else.
  5. Repeat steps 2-4 as necessary.
